## 😔 "문의가 거의 없어요..." — 사장님이 잘못 생각하고 있었습니다

○○유통 사장님은 월 8건의 문의를 "원래 이렇게 적은 거겠지"라고 생각하셨습니다. 홈페이지보다는 영업력이나 제품의 문제라고 보셨죠.

그런데 분석 결과는 달랐습니다. **문제의 원인은 바로 홈페이지였습니다.**

리뉴얼 3개월 후, 월 문의가 8건에서 **26건으로 3.2배** 늘었습니다. 어떻게 가능했을까요?

---

## 🔍 리뉴얼 전 상황 — 숫자가 말해줍니다

○○유통은 2014년에 제작한 PHP 기반 홈페이지를 그대로 사용하고 있었습니다. 저희가 분석한 결과는 이랬습니다.

> 💡 **PHP란?** 홈페이지를 만드는 오래된 프로그래밍 언어입니다. 2014년에는 많이 쓰였지만, 현재 기준으로는 속도와 보안 면에서 많이 뒤처져 있어요.

| 문제 | 수치 | 업계 평균 |
|------|------|---------|
| 📉 PageSpeed 점수 (모바일) | **38점** | 72점 |
| 🚪 이탈률 | **72%** | — |
| ❌ 고객 원하는 정보 없음 | 납기·단가·사양 전무 | — |

> 🎯 **핵심 포인트**: 방문자 10명 중 7명이 3초 안에 사이트를 떠나고 있었습니다. 가게에 들어온 손님이 바로 나가버리는 것과 같아요!

## 🎯 리뉴얼 방향 — 고객이 원하는 것부터

고객 인터뷰 3건을 통해 구매 결정자가 가장 먼저 확인하는 정보를 파악했습니다.

1. 📅 **납기 일정** — "언제까지 받을 수 있나요?"
2. 📦 **최소 주문량** — "소량도 되나요?"
3. 🏆 **신뢰 증거** — "어디에 납품하고 있나요?"

이 3가지를 첫 화면에 배치하는 것이 리뉴얼의 핵심이었습니다.

---

## ⚙️ 기술적 개선

```
Before: PHP + jQuery (2014년 기술)
After:  Next.js 15 + Tailwind CSS (2024년 최신 기술)
```

> 💡 **Next.js, Tailwind CSS란?** 현재 세계에서 가장 빠르고 효율적인 홈페이지 개발 도구들입니다. 속도, 보안, 모바일 최적화 모두 훨씬 우수해요.

- **🖼️ 이미지 최적화**: WebP 변환 + 지연 로딩 적용 → LCP 4.2초 → 1.1초
  > 💡 WebP는 JPG보다 훨씬 작은 파일 크기로 같은 품질을 유지하는 이미지 형식입니다.
- **✂️ 코드 스플리팅**: 불필요한 JS 제거 → FID 320ms → 45ms
- **📱 모바일 최적화**: 버튼 크기, 폼 UX 전면 개편

## 🏆 결과 (3개월 후)

| 지표 | Before | After | 변화 |
|------|--------|-------|------|
| 📊 PageSpeed (모바일) | 38점 | **91점** | +140% 🚀 |
| 🚪 이탈률 | 72% | **41%** | -43% ✅ |
| 📞 월 문의 건수 | 8건 | **26건** | **+225%** 🎉 |
| 💼 상담 → 계약 전환 | 12% | **19%** | +58% 📈 |

> 🎊 문의 건수는 3.2배, 계약 전환율도 함께 올라 매출 효과는 그 이상이었습니다!
