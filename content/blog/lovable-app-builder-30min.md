## "견적서 보고 포기했어요" — 사업 아이디어가 묻혀버린 순간

사업 아이디어는 분명히 있었습니다.

단골 손님 예약을 관리하는 앱, 직원별 주문 현황을 한눈에 보는 화면, 고객이 직접 메뉴를 고르는 모바일 주문 페이지. 머릿속에는 다 그려져 있었는데, 개발업체에 견적을 물어보니 "기본 기능만 해도 최소 500만 원, 유지보수는 월 별도"라는 말이 돌아왔습니다. 두 번 전화했다가 두 번 다 포기했습니다.

2026년, 그 상황이 바뀌었습니다.

## Lovable이란? — "말로 설명하면 앱이 나온다"

**Lovable**([lovable.dev](https://lovable.dev))은 자연어 채팅만으로 풀스택 웹앱을 만들어주는 AI 서비스입니다.

쉽게 말하면 이렇습니다. 개발자에게 "예약 날짜 선택하면 문자 자동 발송되는 기능 넣어주세요"라고 말하는 대신, **Lovable 채팅창에 그대로 입력하면 앱이 만들어집니다.** 코딩을 전혀 몰라도 됩니다. 한국어 프롬프트도 지원합니다.

Lovable은 2024년 말 출시 이후 2개월 만에 연간 반복 매출(ARR) 2,000만 달러(약 270억 원)를 달성했습니다. 유럽 스타트업 역사상 가장 빠른 기록입니다. 비개발자들이 그만큼 이 도구에 몰렸다는 뜻입니다.

만들어진 앱은 **프로토타입이 아닙니다.** 실제 데이터베이스(Supabase)가 자동으로 연결되고, 인터넷에 바로 배포 가능한 주소도 생성됩니다.

## Lovable vs 외주 개발 — 솔직한 비교

![Lovable과 외주 개발 비용·속도 비교](/images/blog/lovable-app-builder-30min-fig1.svg)

*▲ Lovable 무료 플랜 vs 일반 외주 개발 비교 · 출처: lovable.dev 공식 플랜*

**Lovable이 유리한 경우:**
- 아이디어를 빠르게 시험해보고 싶은 경우 (MVP 검증)
- 예약·주문·고객 관리처럼 **정형화된 기능**이 필요한 소상공인
- 개발 예산이 없거나 당장 만들어볼 도구가 필요한 경우

**외주 개발이 여전히 필요한 경우:**
- 기존 POS 시스템·배달 플랫폼과 **API 연동**이 필요한 경우
- 초당 수천 건 처리하는 **대용량 트래픽** 환경
- 앱스토어 등록이 필요한 **네이티브 모바일 앱**

Lovable의 무료 플랜은 **하루 5회 프롬프트 요청**이 제한입니다. 큰 기능 하나를 하루에 5번 안에 완성하는 방식으로 쓰면 됩니다. 유료 플랜은 월 25달러(약 3만 5천 원)부터 시작합니다.

대안 도구로 **Bolt.new**([bolt.new](https://bolt.new))도 있습니다. 개발 경험이 조금 있거나 코드를 직접 수정하고 싶은 분에게 더 친화적이고, 무료 플랜이 있습니다.

> **비젼솔루션이 보는 관점:** 도구의 진짜 가치는 "할 수 있는 것"이 아니라 "처음으로 해볼 수 있게 되는 것"에 있다. Lovable의 의미는 500만 원짜리 앱을 무료로 만드는 게 아니라, 아이디어를 가진 사람이 검증할 수 있는 기회를 처음으로 갖게 됐다는 데 있다. 검증된 아이디어는 그때 제대로 투자해도 늦지 않는다.

## 소상공인 실전 시나리오 — 미용실 예약 앱 30분 완성

경기도의 한 미용실 원장 A씨를 예로 들어보겠습니다.

A씨는 **카카오톡 메시지로 예약을 받다가 중복 예약, 노쇼, 담당자 부재 시 누락 문제**가 반복됐습니다. 직원들이 예약 확인에만 하루 1시간 이상 소비했습니다.

Lovable에 이렇게 입력했습니다:

> "미용실 예약 앱을 만들어주세요. 고객이 날짜와 시간, 담당 디자이너를 선택할 수 있고, 예약이 완료되면 확인 화면이 나오는 앱입니다."

30분 후, 날짜 선택·담당자 배정·예약 목록 관리 화면이 있는 웹앱이 완성됐습니다. Supabase 데이터베이스가 자동 연결되어 예약 데이터가 실시간으로 저장됩니다. 직원은 같은 앱의 관리자 화면에서 오늘 예약을 확인합니다.

## 30분 실습 — 지금 바로 따라하기

준비물: 컴퓨터(사양 무관), 인터넷, 이메일 계정 하나.

**1단계 — 가입**

[lovable.dev](https://lovable.dev) 접속 → 우측 상단 "Sign up" → 구글 계정 또는 이메일로 가입. 무료 플랜 자동 적용.

**2단계 — 첫 프롬프트 입력 (아래 복붙)**

```
소상공인 고객 관리 앱을 만들어주세요.
기능: 고객 이름·연락처·방문일 입력, 목록 조회, 검색.
디자인: 모바일 화면에서 보기 좋게.
```

한국어로 입력해도 됩니다. Lovable이 코드를 자동 생성합니다.

**3단계 — 미리보기 확인**

오른쪽 패널에 앱 미리보기가 실시간으로 나타납니다. 마음에 안 드는 부분은 채팅창에 "버튼 색을 파란색으로 바꿔주세요"처럼 추가 요청합니다.

**4단계 — 배포**

우측 상단 "Publish" 버튼 클릭 → 공개 URL 생성 완료. 이 주소를 카카오톡이나 인스타그램 링크로 고객에게 공유하면 됩니다.

**5단계 — 데이터베이스 확인 (선택)**

Supabase 연동을 활성화하면 입력된 데이터가 클라우드에 자동 저장됩니다. Lovable 내에서 "Supabase 연결" 버튼 클릭 후 안내를 따라하면 됩니다.

## 한계도 알아야 합니다

Lovable은 빠르지만 **한계가 있습니다.**

- 복잡한 결제 시스템, 카드 단말기 연동은 별도 작업 필요
- 앱을 앱스토어에 올리는 것은 불가 (웹앱 형태만 지원)
- 하루 5회 무료 제한 — 기능이 많아지면 유료 전환 필요
- 생성된 코드의 보안 검토는 직접 해야 함 (운영 전 전문가 확인 권장)

무료로 아이디어를 검증하고, 실제 운영이 결정되면 그때 더 정밀하게 다듬는 순서가 맞습니다.
