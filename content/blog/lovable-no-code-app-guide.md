## 앱 견적서 받고 포기한 적 있으신가요?

사업 아이디어는 분명히 있었습니다.

예약 시스템을 만들고 싶었던 미용실 원장님, 단골 고객 포인트 관리 앱이 필요했던 카페 사장님, 주문을 자동으로 받는 시스템을 원했던 식당 운영자. 모두 같은 벽에 부딪혔습니다.

"최소 500만 원부터 시작하고요, 유지비는 별도입니다."

개발업체에서 돌아오는 견적은 항상 비슷했습니다. 500만 원, 많게는 2,000만 원. 거기에 매달 유지보수 비용까지. 결국 대부분은 아이디어 노트를 서랍 속에 넣어두고 잊었습니다.

2026년, 그 서랍을 다시 꺼낼 때가 됐습니다.

## Lovable — 몰랐던 사실

**AI 앱 빌더 Lovable**이 2026년 초 공식 출시 2개월 만에 연간 반복 매출(ARR) **2,000만 달러(약 270억 원)**를 달성했습니다. 유럽 스타트업 역사상 최단 기록입니다.

왜 이렇게 빠르게 성장했을까요? 이유는 단순합니다. **코딩을 전혀 몰라도 실제 작동하는 앱을 만들 수 있기 때문**입니다.

Lovable의 핵심 특징을 정리하면 이렇습니다:

- **자연어 프롬프트로 풀스택 웹앱 생성** — "예약 받는 앱 만들어줘"라고 입력하면 그게 곧 개발 지시서가 됩니다
- **한국어 프롬프트 지원** — 영어를 몰라도 됩니다
- **Supabase DB 자동 연동** — 데이터베이스 구성까지 자동으로 처리
- **프로토타입이 아닌 실제 배포 가능한 앱** — 만들고 나면 바로 고객에게 공유할 수 있습니다
- **무료 플랜 제공** — 하루 5회 프롬프트 요청은 무료

## 30분 실습 — 예약 앱 만들기 순서

실제로 어떻게 진행되는지 단계별로 따라가 보겠습니다. 예시는 '헤어샵 예약 앱'입니다.

### 1단계: 가입 (3분)

[lovable.dev](https://lovable.dev)에 접속해 구글 계정으로 로그인합니다. 별도 설치 없이 브라우저에서 모두 진행됩니다.

### 2단계: 프롬프트 입력 (2분)

새 프로젝트를 만들고 다음과 같이 입력합니다:

> "헤어샵 예약 시스템을 만들어줘. 고객이 이름, 연락처, 원하는 날짜와 시간을 입력하면 예약이 저장되고, 원장은 예약 목록을 볼 수 있어야 해."

한국어로 입력해도 됩니다. 복잡한 문법은 필요 없고, 하고 싶은 말을 자연스럽게 쓰면 됩니다.

### 3단계: AI가 앱을 생성 (5~10분)

Lovable이 자동으로 다음을 처리합니다:

- 화면 레이아웃 설계
- 예약 데이터를 저장할 데이터베이스 구성 (Supabase 자동 연동)
- 원장 전용 관리 화면 생성
- 반응형 디자인 적용 (PC·모바일 모두 작동)

### 4단계: 수정 요청 (5분)

결과가 마음에 들지 않는 부분이 있다면 다시 말하면 됩니다:

> "예약 확인 버튼 색깔을 초록색으로 바꿔줘"
> "날짜 선택 달력을 더 크게 만들어줘"

채팅하듯 대화하면 실시간으로 수정됩니다.

### 5단계: 배포 (1분)

마음에 들면 **Share** 버튼 하나로 인터넷에 공개할 수 있습니다. 도메인 주소가 자동 생성되어 즉시 고객에게 링크를 공유할 수 있습니다.

## Bolt.new — 개발자 성향이라면 이쪽도

비슷한 도구로 **Bolt.new**도 있습니다. Lovable보다 기술적인 제어가 가능해 개발 경험이 있는 분에게 더 적합합니다. 두 도구 모두 무료 플랜을 제공하니 직접 비교해 보는 것도 좋습니다.

| 항목 | Lovable | Bolt.new |
|------|---------|---------|
| 주요 대상 | 비개발자, 소상공인 | 개발자, 기술 친숙 사용자 |
| 한국어 지원 | ✅ | ✅ |
| 무료 플랜 | 하루 5회 | 제한적 |
| DB 연동 | Supabase 자동 | 수동 설정 가능 |

## 그래서 어디까지 가능한가

Lovable로 만들 수 있는 것들:

- **예약 시스템** — 헤어샵, 네일샵, 학원, 상담소
- **주문 관리 앱** — 식당, 카페, 온라인 셀러
- **고객 관리 DB** — 단골 포인트, 방문 기록
- **설문·신청서** — 이벤트 참가, 입점 문의
- **소규모 쇼핑몰** — 제품 목록, 장바구니

단, 결제 모듈(PG사 연동), 복잡한 재고 관리, 대규모 동시 접속 처리 같은 기능은 별도 구성이 필요합니다. 이런 경우라면 전문 개발팀의 도움이 더 효율적입니다.
