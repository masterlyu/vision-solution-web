새벽 7시, 출근 전 커피를 내리는 A씨(43세, 중소기업 마케팅 담당).

스마트폰을 켜보니 어젯밤에 설정해둔 Claude가 이미 오늘의 블로그 초안을 만들어놨습니다. 트렌드 키워드도 정리됐고, SNS 게시글 3개도 준비된 상태였습니다.

A씨가 한 일이라곤 딱 하나. 지난주에 "매일 아침 6시에 이 작업 해줘"라고 설정한 것뿐이었습니다.

이게 가능한 이유가 바로 **하네스 엔지니어링**입니다.

![center](/mascot/md/emotion/cat_happy.png)

## AI를 직접 켜야 하는 게 불편하다면

많은 분이 AI를 이렇게 씁니다. 매번 앱 켜고 → 뭘 시킬지 생각하고 → 직접 입력하고 → 결과 확인하고 → 다음 날 또 반복.

솔직히 말씀드리면, 이 방식은 AI를 제대로 쓰는 게 아니에요. AI가 할 일을 사람이 매번 챙겨줘야 한다면, 그건 자동화가 아닙니다.

진짜 자동화란 이겁니다. 한 번 설정해두면 AI가 스스로 트리거(신호)를 받고, 스스로 일하고, 스스로 결과를 저장하는 것.

그 핵심에 **하네스(Harness)** 시스템이 있습니다.

## 하네스란? — AI에게 고삐를 채운다

하네스는 원래 말(馬)에게 씌우는 마구(馬具)를 뜻합니다.

야생마가 아무리 빠르고 강해도 안장과 고삐가 없으면 원하는 방향으로 달릴 수 없죠. Claude 같은 AI도 마찬가지입니다. AI 모델 자체는 능력이 있지만, 언제 실행할지·무엇을 허용할지·어떤 도구를 쓸지 — 이걸 제어하는 구조가 없으면 실무에서 쓸 수 없어요.

Claude Code(앤트로픽의 AI 개발 도구)에서 이 제어 구조를 **하네스**라고 부릅니다.

> "에이전트를 만드는 것은 모델 선택이 5%, 하네스 설계가 95%다."
> — Claude Code 공식 문서

## 자동화를 만드는 3가지 핵심 도구

### 1. Hooks — AI가 행동하기 전후에 자동 실행

Hooks는 Claude가 특정 행동을 할 때 자동으로 실행되는 스크립트입니다. 총 12개 시점을 지원하는데, 가장 많이 쓰는 3개만 소개할게요.

| Hook 이름 | 언제 실행 | 실용 예시 |
|-----------|-----------|----------|
| SessionStart | 세션 시작 시 | 오늘의 작업 규칙·맥락을 자동 주입 |
| PreToolUse | 도구 실행 직전 | 위험한 명령(rm -rf 등) 자동 차단 |
| PostToolUse | 도구 실행 직후 | 작업 결과 자동 로그 저장 |

설정 방법은 생각보다 간단합니다. Claude Code의 `settings.json` 파일에 한 번만 입력하면 됩니다.

```json
{
  "hooks": {
    "SessionStart": [
      { "command": "cat ~/projects/오늘작업규칙.md" }
    ]
  }
}
```

이 설정 하나면 Claude를 켤 때마다 지정한 규칙이 자동으로 주입됩니다. "매번 같은 설명 반복"에서 해방되는 거죠.

### 2. Routines — 컴퓨터 꺼져도 클라우드에서 돌아간다

2026년 5월 앤트로픽이 공식 출시한 기능입니다(InfoQ 보도). Routines는 클라우드 기반으로 작동하기 때문에 내 노트북이 꺼져있어도 정해진 시간에 Claude가 스스로 작동합니다.

트리거 방식 3가지:
- **Scheduled**: "매일 오전 8시" 같은 시간 기반
- **API**: 외부 시스템이 신호를 보낼 때
- **GitHub**: PR 생성·릴리즈 이벤트에 반응

예를 들어 "매주 월요일 오전 9시에 지난 주 블로그 트래픽 분석 후 다음 주 키워드 추천해줘"를 한 번 설정해두면, 월요일 아침마다 자동으로 분석 결과가 나옵니다.

설정 위치: `claude.ai/code/routines` (데스크탑 앱, CLI에서도 동일)

![center](/mascot/md/emotion/cat_surprised.png)

### 3. /loop — 반복 작업을 3초 만에 등록

Claude Code 내에서 직접 타이핑하는 방식입니다. 가장 빠르게 자동화를 시작할 수 있어요.

```
/loop 1d 어제 수집된 트렌드 키워드로 SNS 게시글 초안 3개 작성 후 drafts/ 폴더에 저장
```

이 한 줄이면 됩니다. "1d"는 하루 간격. Claude가 자동으로 스케줄에 등록하고 매일 실행합니다. 기본적으로 3일 후 자동 만료되어 무한 실행을 방지합니다.

## 실전 활용 사례 5가지

| 자동화 항목 | 사용 도구 | 효과 |
|------------|---------|------|
| 매일 아침 블로그 초안 작성 | Routines (daily) | 주 5일 × 1시간 절약 |
| GitHub PR 생성 시 자동 코드 리뷰 | GitHub 트리거 Routine | 리뷰 대기 시간 제로 |
| 세션 시작 시 프로젝트 맥락 자동 주입 | SessionStart Hook | 매 세션 같은 설명 반복 제거 |
| 위험 명령어 자동 차단 | PreToolUse Hook | 실수로 인한 데이터 삭제 방지 |
| 코딩 후 자동 린터 실행 | PostToolUse Hook | 코드 품질 유지 자동화 |

## 시작 전 주의할 점 하나

하네스 설정에서 가장 흔한 실수는 "너무 많은 걸 한꺼번에 자동화하려는 것"입니다.

처음엔 딱 하나만 하세요.

예를 들어 "매일 아침 어제 이슈를 정리해서 오늘 할 일 목록 만들어줘" 하나만요. 이게 잘 돌아가면 그다음 자동화를 추가하는 겁니다. Hook 10개, Routine 5개를 한 번에 설정하면 뭐가 잘못됐는지 파악하기 어렵습니다.

## 지금 바로 시작할 수 있는 첫 번째 하네스

1. Claude Code 설치 (무료, `npm install -g @anthropic-ai/claude-code`)
2. `~/.claude/settings.json` 파일 생성
3. SessionStart Hook에 오늘의 작업 규칙 파일 경로 입력
4. Claude Code 실행 → 자동으로 규칙 주입되는 것 확인

이 4단계면 첫 번째 자동화가 완성됩니다. 이후에 Routines와 /loop을 추가하면서 점차 확장하시면 됩니다.
