> 홈페이지가 있는데 문의가 없다. 어디가 문제인지 모르겠다.
> 이런 상황이라면, 지금 바로 읽어보세요.

홈페이지를 운영하는 분들이 가장 많이 하시는 말씀이 있습니다.

**"우리 홈페이지, 그냥 있긴 한데… 뭐가 문제인지 모르겠어요."**

이걸 몰라서 생기는 문제가 생각보다 훨씬 큽니다. 광고비는 쓰는데 문의는 없고, 홈페이지를 고치고 싶은데 뭘 어디서부터 손봐야 할지 막막하죠.

저희 비전솔루션이 만든 **홈페이지 운영상태진단** 서비스는 이 고민을 해결하기 위해 만들었습니다. URL 하나만 입력하면 AI가 자동으로 분석하고, 48시간 안에 PDF 리포트를 이메일로 발송합니다.

오늘은 실제 진단 결과물을 그대로 공개합니다. 리포트에 나오는 항목이 무슨 뜻인지, 초보자도 이해할 수 있도록 하나하나 설명드리겠습니다.

## 종합 등급이란? — A~F까지 5단계
![right](/mascot/md/emotion/cat_cheer.png)

리포트 오른쪽 상단에 크게 표시되는 **등급**은 홈페이지의 전반적인 상태를 한눈에 나타냅니다.

| 등급 | 점수 | 의미 |
|------|------|------|
| A | 90점 이상 | 현재 상태 우수. 정기 모니터링 권장 |
| B | 80~89점 | 양호. 일부 개선으로 상위 도달 가능 |
| C | 70~79점 | 보통. 주요 항목 개선 필요 |
| D | 60~69점 | 미흡. 구조적 개선이 필요한 상태 |
| F | 60점 미만 | 위험. 빠른 조치 권장 |

등급이 낮다고 해서 당장 큰일 나는 건 아닙니다. 다만 **방문자가 불편을 느끼고 있을 가능성이 높다**는 신호입니다.

## 사이트 유형 판단 — AI가 홈페이지 성격을 자동 분류합니다
![right](/mascot/md/emotion/cat_cheer.png)

리포트 상단에 **"중소 IT·웹 서비스업 / 신뢰도 47% · IT서비스형 프로파일 적용"** 이라고 나옵니다.

이게 무슨 뜻이냐면, AI가 홈페이지 내용을 보고 "이 사이트는 어떤 업종, 어떤 규모로 운영되는 홈페이지인가"를 자동으로 추측한다는 뜻입니다.

업종마다 홈페이지에서 중요한 요소가 다릅니다. 쇼핑몰은 결제 편의성이 중요하고, IT 회사는 신뢰도와 기술력 표현이 중요하죠. 이 유형 판단을 바탕으로 진단 기준이 조금씩 달라집니다.

**신뢰도 47%**는 AI가 "이 사이트가 IT·웹 서비스업일 가능성이 47%"라고 판단했다는 뜻입니다. 100%가 아닌 이유는 공개된 정보만으로 분석하기 때문입니다.

## 3축 진단 점수 — 100점 만점, 세 가지로 나눕니다
![right](/mascot/md/emotion/cat_cheer.png)

점수는 크게 세 가지 축으로 나뉩니다. 합산하면 100점 만점입니다.

### 기술 기반 (35점 만점)

**쉽게 말하면:** 홈페이지가 "기술적으로 제대로 만들어졌는가"를 봅니다.

주로 이런 것들을 확인합니다:
- **HTTPS 적용** — 주소창에 자물쇠 아이콘이 있나요? 없으면 방문자 정보가 노출될 수 있습니다.
- **보안 헤더 설정** — 브라우저에게 "이 사이트는 안전하게 사용해"라고 알려주는 기술 설정입니다. 없으면 해킹 시도에 더 취약합니다.
- **메타태그 최적화** — 검색 결과에서 어떻게 보이는지를 결정하는 텍스트입니다. 잘 작성돼야 검색 노출이 잘 됩니다.
- **robots.txt** — 검색 엔진에게 "이 페이지는 수집해도 돼, 이건 하지 마"를 알려주는 파일입니다.
- **사이트맵** — 홈페이지의 전체 페이지 목록을 정리한 파일로, 검색 엔진이 빠르게 페이지를 발견하게 도와줍니다.

visionc.co.kr은 **35점 만점**을 받았습니다. 기술적인 기반이 탄탄하다는 뜻입니다.

### 사용자 경험 (35점 만점)

**쉽게 말하면:** 홈페이지를 방문한 사람이 "편하게 쓸 수 있는가"를 봅니다.

확인 항목 예시:
- **모바일 반응형** — 스마트폰에서 봤을 때 글자가 너무 작거나 버튼이 짤리지 않는지
- **뷰포트 설정** — 모바일 화면 크기에 맞게 콘텐츠가 조정되는지
- **터치 요소 크기** — 손가락으로 클릭하기 충분히 큰지 (너무 작으면 오타가 나서 불편합니다)
- **폰트 가독성** — 글자가 읽기 편한 크기와 색상인지

visionc.co.kr은 **35점 만점**이지만, 아래에서 보시다시피 개선 권장 항목이 발견됐습니다.

### 현대 기준 (30점 만점)

**쉽게 말하면:** "지금 시대에 기본으로 갖춰야 할 것들이 있는가"를 봅니다.

확인 항목 예시:
- **GA4 연동** — 방문자 통계 도구가 설치돼 있는지 (없으면 누가 왔는지 알 수 없습니다)
- **페이지 로딩 속도** — 3초 이상 걸리면 방문자의 절반이 떠납니다
- **챗봇 또는 상담 도구** — 방문자가 바로 질문할 수 있는 창구가 있는지
- **스키마 마크업** — 검색 결과에서 별점·FAQ 등 풍부한 정보가 표시되게 하는 코드
- **CTA 배치** — "문의하기", "상담 신청" 버튼이 눈에 잘 띄는 위치에 있는지

visionc.co.kr은 **27점/30점**으로, 개선 여지가 조금 있었습니다.

## 진단 결과 요약 카드 — 숫자로 한눈에

리포트에는 4개의 요약 카드가 있습니다:

- **97점 / 100점 만점** — 종합 점수
- **1건 / 즉시 개선** — 지금 당장 고쳐야 할 것 (방치하면 문의 손실)
- **5건 / 개선 권장** — 시간 여유가 있을 때 개선하면 좋은 것
- **23건 / 양호** — 현재 기준에서 문제없는 항목 수

"즉시 개선"과 "개선 권장"의 차이는 **긴급도**입니다. 즉시 개선은 지금 당장 방문자 이탈이나 신뢰 손실이 일어나고 있는 상황, 개선 권장은 더 잘할 수 있는 기회입니다.

## 기술 스택 추정 — AI가 홈페이지 기술을 알아냅니다

리포트 하단에 이런 표가 있습니다:

| 구분 | 내용 | 상태 |
|------|------|------|
| 웹서버 | Cloudflare | 확인됨 |
| 호스팅 | Vercel | 확인됨 |
| 프레임워크 | Next.js (React 기반 SSR/SSG) | 확인됨 |
| 런타임 | Node.js | 추정됨 |
| OS | Linux (Node.js 환경) | 추정됨 |

**웹서버**는 홈페이지 파일을 인터넷에 전달해주는 역할을 합니다. Cloudflare는 전 세계 어디서 접속해도 빠르고 보안이 강력한 서비스입니다.

**호스팅**은 홈페이지 파일이 실제로 저장되는 곳입니다. Vercel은 빠른 배포와 글로벌 속도로 유명합니다.

**프레임워크**는 홈페이지를 만들 때 사용한 기술 도구입니다. Next.js는 React 기반으로 속도와 SEO에 강한 최신 기술입니다.

**"확인됨"**은 AI가 공개 정보로 직접 확인한 것, **"추정됨"**은 간접적으로 유추한 것입니다.

이 정보가 왜 필요하냐면, **리뉴얼이나 유지보수를 맡길 업체를 선정할 때** 현재 기술 스택을 알아야 적합한 개발자를 찾을 수 있기 때문입니다.

## 항목별 상세 진단 — 뭘 고쳐야 하는지 구체적으로 알 수 있습니다

진단 리포트의 핵심입니다. 각 축별로 **어떤 항목이 문제인지**, **왜 문제인지**, **어떻게 고치면 되는지**까지 알려줍니다.

### 사용자 경험 — 발견된 개선 항목 3가지

**1. 문의·연락 버튼 접근성 (영향 높음)**

- 현재 상태: 전화 링크 없음
- 개선 방향: 전화 링크(tel:)·문의 폼을 첫 화면에서 스크롤 없이 접근 가능하게 배치

**쉽게 말하면:** 방문자가 홈페이지에 들어왔을 때 바로 "전화하기" 버튼이나 "문의하기" 링크가 보여야 합니다. 스크롤을 내려야만 찾을 수 있다면, 많은 사람이 그냥 나가버립니다. 특히 모바일에서는 전화번호를 탭하면 바로 전화가 걸리는 `tel:` 링크가 필수입니다.

**2. 하단 연락처 표시 (영향 중간)**

- 현재 상태: 하단에 연락처·사업자 정보 없음
- 개선 방향: 전화번호·이메일·주소를 하단에 명시

**쉽게 말하면:** 홈페이지 가장 아래쪽(footer)에 회사 전화번호, 이메일, 주소, 사업자등록번호가 표시돼야 합니다. 이게 없으면 "이 회사 실제로 존재하는 건지 모르겠다"는 의심을 받습니다. 전자상거래법상 의무 표기 항목이기도 합니다.

**3. 웹 접근성 기초 (영향 높음)**

- 현재 상태: 건너뛰기 링크 없음, 키보드 탐색 불편
- 개선 방향: 건너뛰기 링크 + ARIA 레이블 추가

**쉽게 말하면:** 시각 장애인이나 키보드만 사용하는 분들도 홈페이지를 이용할 수 있어야 합니다. "건너뛰기 링크"는 키보드로 접속했을 때 메뉴를 일일이 탭하지 않고 바로 본문으로 이동하게 해주는 기능입니다. 공공기관이나 일정 규모 이상의 기업에서는 법적 의무 항목이기도 합니다.

### 현대 기준 — 발견된 개선 항목 3가지

**1. 기술 스택 현대성 (영향 높음)**

- 현재 상태: Next.js 기반 — 지속적 업데이트 관리 필요
- 개선 방향: 최신 버전 업데이트로 보안·성능·유지보수성 개선

**쉽게 말하면:** 홈페이지를 만든 기술(프레임워크)도 계속 업데이트됩니다. 오래된 버전을 그대로 쓰면 보안 취약점이 생기고, 속도가 느려지고, 나중에 수정하기가 더 어려워집니다. 정기적으로 업데이트해주거나, 유지보수 계약을 통해 관리받는 것이 좋습니다.

**2. SNS·채널 연동 (영향 낮음)**

- 현재 상태: SNS·소셜 채널 링크 없음
- 개선 방향: 인스타그램·유튜브·카카오 채널 링크 추가

**쉽게 말하면:** 홈페이지에서 인스타그램, 유튜브, 카카오 채널 같은 SNS로 연결되는 링크가 없다는 뜻입니다. SNS 링크가 있으면 방문자가 팔로우하거나 구독해서 나중에 다시 찾아올 가능성이 높아집니다. 브랜드 노출 기회가 늘어나는 것이죠.

**3. 쿠키·개인정보 동의 배너 (영향 중간)**

- 현재 상태: 쿠키 동의 배너 없음
- 개선 방향: 쿠키 동의 팝업 추가 → 개인정보보호법 대응

**쉽게 말하면:** 방문자의 개인정보(방문 기록, 광고 추적 등)를 수집할 때는 반드시 동의를 받아야 합니다. 홈페이지에 들어갔을 때 "쿠키 사용에 동의하시겠어요?" 같은 팝업이 뜨는 게 그겁니다. 개인정보보호법에 따라 이게 없으면 과태료 대상이 될 수 있습니다.

## 리포트를 받은 후 어떻게 해야 하나요?

**점수가 높다면(A·B등급):** 현재 상태가 양호합니다. 발견된 개선 권장 항목을 하나씩 정리해 적용하면 됩니다.

**점수가 낮다면(C등급 이하):** 즉시 개선 항목부터 시작하세요. 혼자 하기 어렵다면 전문가 상담을 받으시는 게 빠릅니다.

리포트에는 **견적이 포함되지 않습니다.** 결과를 보고 필요하다고 느끼실 때 별도로 상담하시면 됩니다. 부담 없이 먼저 현황만 파악해 보세요.
