매일 아침 8시, A씨의 화면에는 이미 오늘치 블로그 초안이 올라와 있습니다. A씨는 어젯밤 잠든 사이에 아무것도 하지 않았습니다. 그런데 AI가 혼자 트렌드를 수집하고, 초안을 쓰고, 저장까지 마쳤습니다.

"혹시 나도 이렇게 할 수 있을까?"

가능합니다. 이것이 바로 **하네스 엔지니어링(Harness Engineering)** 입니다.

---

## AI를 야생마에서 일꾼으로 — 하네스란?

AI는 혼자서는 방향을 모릅니다. 질문에는 잘 대답하지만, 알아서 매일 정해진 일을 하진 않습니다. '말 걸어야 움직이는 AI'가 기본값이에요.

하네스(Harness)는 원래 말(馬)에 씌우는 마구(馬具)입니다. 아무리 빠른 야생마도 안장과 고삐 없이는 원하는 곳으로 달릴 수 없습니다. AI 하네스는 이 고삐 역할입니다.

![center](/mascot/md/emotion/cat_happy.png)

Claude Code의 하네스가 제공하는 것은 세 가지입니다. 언제 실행할지(스케줄), 무엇을 하지 말아야 하는지(안전 게이트), 결과를 어디에 저장할지(자동 처리). 한 마디로, AI에게 **반복 업무를 맡겨놓고 잊어버릴 수 있게** 해주는 구조입니다.

에이전트를 만드는 것은 "5% 모델 루프 + 95% 하네스"라는 말이 있습니다. 실제로 AI 자동화에서 어려운 부분은 AI 자체가 아니라, 이 하네스를 설계하는 일입니다.

---

## 실전 도구 1 — 훅(Hooks)으로 AI 행동 제어하기

훅(Hooks)은 Claude Code가 무언가를 하기 **전후에 자동으로 실행되는 명령어**입니다. 쉽게 말하면, AI가 특정 행동을 할 때 끼어들 수 있는 고리(Hook)를 만드는 거예요.

예를 들어 이런 게 가능합니다. 세션이 시작되면 오늘 할 일 목록 자동 주입, AI가 파일을 수정하려 하면 위험 명령어인지 먼저 확인, 작업이 끝나면 결과를 슬랙으로 자동 발송.

![center](/mascot/md/emotion/cat_thinking.png)

실제로 써먹는 훅 설정 예시를 알려드릴게요.

**SessionStart — 세션 시작 시 규칙 자동 주입**

`~/.claude/settings.json` 파일을 열고 아래를 추가합니다:

```json
{
  "hooks": {
    "SessionStart": [
      { "command": "cat ~/projects/my-rules.md" }
    ]
  }
}
```

이제 Claude Code를 켤 때마다 `my-rules.md` 안의 내용이 AI 컨텍스트에 자동으로 들어갑니다. 매번 "이 프로젝트는 이렇게 해줘"를 타이핑할 필요가 없어요.

**PreToolUse — 위험 명령어 자동 차단**

```bash
#!/bin/bash
# hooks/block_dangerous.sh
if echo "$CLAUDE_TOOL_INPUT" | grep -q "rm -rf"; then
  echo "위험한 명령어입니다. 실행을 차단합니다." >&2
  exit 2
fi
```

`exit 2`를 반환하면 AI가 해당 명령을 실행하지 않습니다. 이 차단은 심지어 `bypassPermissions` 모드에서도 우회 불가능합니다. 진짜 정책 강제 수단입니다. Claude Code 훅은 현재 총 12개 라이프사이클 이벤트를 지원합니다.

## 실전 도구 2 — 루틴(Routines)으로 무인 자동화

훅이 '한 세션 안의 제어'라면, 루틴은 **노트북이 꺼져 있어도 돌아가는 클라우드 자동화**입니다.

2026년 5월 Anthropic이 공식 출시한 Routines는 Anthropic의 클라우드 인프라에서 실행됩니다. 내 컴퓨터가 꺼져 있어도 됩니다. 여행 중에도 됩니다.

![center](/mascot/md/emotion/cat_surprised.png)

루틴 트리거는 3가지입니다:

| 트리거 종류 | 언제 실행 | 사용 예 |
|------------|---------|--------|
| Scheduled | 매일/매주/매시간 | 매일 오전 8시 블로그 초안 생성 |
| API | HTTP 요청 받으면 | 버튼 클릭 시 자동 분석 실행 |
| GitHub | PR·릴리즈 이벤트 | 코드 올리면 자동 리뷰 실행 |

**루틴 설정 방법 (3단계):**

1. `claude.ai/code/routines` 접속
2. 프롬프트 입력: "어제 수집된 키워드로 블로그 초안을 drafts/ 폴더에 저장해줘"
3. 트리거 설정: Scheduled → 매일 오전 8시 → 저장소 연결

완료입니다. 이제 매일 아침 자동으로 실행됩니다. 한 번 설정하면 건드릴 필요가 없습니다.

---

## 실전 도구 3 — /loop 명령으로 바로 시작하기

가장 빠른 자동화 방법은 `/loop` 명령입니다. 별도 설정 파일 없이 Claude Code 안에서 바로 씁니다:

```
/loop 1d 어제 수집된 트렌드 키워드로 블로그 초안 작성 후 drafts/ 폴더에 저장
```

`1d`를 하루 1번 주기로 인식해서 cron 스케줄을 자동 등록합니다. 기본적으로 **3일 안전 만료**가 적용됩니다. 무한히 돌다가 서버 자원을 소모하는 것을 막아줍니다.

![center](/mascot/md/emotion/cat_cheer.png)

솔직히 말씀드리면, 처음 써본 사람들이 가장 많이 하는 실수가 있습니다. 루틴 설정 후 "왜 안 되지?" 하는 경우인데요. 저장소 연결(Repository) 설정을 빠뜨린 경우가 대부분입니다. 루틴이 파일을 읽고 써야 할 저장소를 반드시 연결해 주세요.

## 오늘 당장 시작하는 3단계 실행 순서

처음이라면 이 순서를 추천합니다.

**1단계 — SessionStart 훅 먼저**
`~/.claude/settings.json`에 SessionStart 훅만 설정합니다. 매번 타이핑하던 프로젝트 규칙 주입을 자동화하는 것부터 시작하세요. 파일 하나 수정이면 5분 안에 끝납니다.

**2단계 — /loop으로 반복 업무 테스트**
이미 반복하고 있는 작업 하나를 골라 `/loop` 명령으로 등록해보세요. "매일 오전에 이 폴더 정리해줘" 같은 단순한 것부터 시작하면 됩니다.

**3단계 — Routines로 완전 무인화**
익숙해지면 Routines로 넘어갑니다. 내 컴퓨터가 꺼져 있어도, 심지어 여행 중에도 AI가 일을 하고 있습니다.

설정 자체는 어렵지 않습니다. 파일 하나 수정하고, 명령어 한 줄 입력하면 됩니다. 어려운 건 "뭘 자동화할지" 결정하는 것이에요. 그 부분이 막힌다면 아래로 편하게 문의하세요.
