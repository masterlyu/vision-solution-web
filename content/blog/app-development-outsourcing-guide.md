## "앱 만들려다 수천만 원 날린 사례"가 왜 이렇게 많을까?

앱 개발 외주를 맡겼다가 실패하는 사례는 생각보다 훨씬 많습니다.

- 계약금만 받고 잠수를 탄 업체
- 1년 걸려 만든 앱이 실제로 쓸 수 없는 수준
- 추가 요구 사항마다 수백만 원씩 추가 청구
- 납품은 받았는데 버그가 너무 많아서 실제로 못 씀

이런 일이 반복되는 이유는 하나입니다 — **사장님이 앱 개발 과정을 모르기 때문에, 불합리한 조건을 걸러내지 못하는 것**입니다.

이 글을 읽고 나면, 계약 전에 스스로 걸러낼 수 있게 됩니다.

---

## 앱 개발 외주, 얼마나 드나요?

먼저 현실적인 비용 범위를 알아야 합니다.

| 앱 유형 | 예상 비용 | 기간 |
|--------|----------|------|
| 간단한 정보 제공형 앱 (메뉴 소개, 위치 안내 수준) | 300~700만 원 | 1~2개월 |
| 예약·주문·결제 기능 포함 | 700~2,000만 원 | 2~4개월 |
| 회원가입·커뮤니티·SNS 기능 포함 | 2,000~5,000만 원 | 4~8개월 |
| 대형 플랫폼·마켓플레이스 | 5,000만 원 이상 | 6개월+ |

"100만 원짜리 앱"을 광고하는 업체가 있습니다. 기능이 극도로 단순하거나, 기존 템플릿에 이름만 넣는 수준입니다. 실제 비즈니스에 쓸 수 있는 앱을 기대하면 안 됩니다.

## 실패하지 않는 업체 선택 기준

### 기준 1: 앱스토어에 실제로 출시된 앱이 있는가?

"저희가 만든 앱입니다"라고 보여주는 포트폴리오가 실제 앱스토어(구글 플레이, 애플 앱스토어)에 있는지 확인하세요.

앱스토어에 올라와 있다면:
- 다운로드 수 확인 (실제 운영 중인가?)
- 리뷰 확인 (버그가 많진 않은가?)
- 마지막 업데이트 날짜 확인 (유지보수를 하고 있는가?)

---

### 기준 2: 요구사항을 문서로 정리해주는가?

좋은 업체는 계약 전에 **기능 명세서(요구사항 정의서)** 를 작성합니다. 이 문서에는 앱에 들어가는 모든 화면과 기능이 목록으로 적혀 있습니다.

이 문서가 없으면 나중에 "그건 말씀하신 범위가 아니에요 — 추가 비용이 필요합니다"라는 말을 듣게 됩니다.

---

### 기준 3: iOS와 Android 동시 개발인가, 한 플랫폼인가?

같은 앱을 iOS(아이폰)와 Android(갤럭시) 둘 다 만드는 건 비용이 더 들 수 있습니다. 단, **React Native나 Flutter** 같은 크로스플랫폼 기술을 쓰면 하나의 코드로 둘 다 만들어 비용을 절반으로 줄일 수 있습니다.

업체에게 어떤 기술을 쓰는지 반드시 물어보세요.

---

### 기준 4: 소스 코드 소유권은 내가 갖는가?

앱 개발이 완료되면 소스 코드(앱의 설계도)를 넘겨받아야 합니다. 이게 없으면 업체를 바꾸고 싶어도 처음부터 다시 만들어야 합니다.

계약서에 반드시 명시: **"납품 완료 시 소스 코드 전체를 발주처에 이관한다"**

---

### 기준 5: 출시 후 유지보수 계획이 있는가?

앱은 출시 후가 진짜 시작입니다. 스마트폰 OS가 업데이트될 때마다 앱이 제대로 작동하는지 확인해야 합니다. 버그가 발견되면 빠르게 수정해야 합니다.

- 출시 후 몇 개월간 무상 수정이 포함되는가?
- 월 유지보수 비용은 얼마인가?
- 긴급 버그 발생 시 몇 시간 내에 대응하는가?

## 계약 전 체크리스트

```
[ ] 실제 앱스토어에 출시된 포트폴리오 확인
[ ] 기능 명세서 작성 여부 확인
[ ] iOS/Android 개발 방식 확인
[ ] 소스 코드 이관 계약서 명시
[ ] 출시 후 유지보수 조건 확인
[ ] 중간 단계 결과물(화면 시안, 테스트 버전) 확인 일정
[ ] 추가 비용 발생 기준 명시
```

## 앱이 정말 필요한가요?

사실 가장 중요한 질문입니다.

앱을 만들기 전에 **"모바일 웹으로 해결되지 않는가?"** 를 먼저 고민하세요. 앱은 설치를 해야 쓸 수 있어서 진입 장벽이 있습니다. 반면 모바일 웹은 링크 하나로 바로 접근할 수 있습니다.

경우에 따라 앱보다 모바일 최적화된 홈페이지가 더 효과적인 경우가 많습니다.
