# Course 02 강의자료 작성 지침 (1편에서 확립 · 2편~11편 공통 적용)

> 1편 '의사결정'을 제작하며 확립한 표준. **새 편을 쓸 때는 이 문서를 먼저 읽고,
> `buildai_part1_data.py`를 템플릿으로 복제해 내용만 교체**한다. 스타일·파이프라인은 자동 적용되므로
> 작성자는 "콘텐츠(슬라이드 텍스트 + 스피커노트)"에만 집중하면 된다.

## 0. 산출물 (편마다 동일)
- `build-ai-part{N}-slides.pptx` — 다크 프리미엄 풀블리드 이미지 덱 (코스1 동일 스타일)
- `build-ai-part{N}-speaker-notes.pdf` — 화이트 에디토리얼 박스 노트카드 (코스1 동일 스타일)
- 비번 게이트 다운로드(`visioncDown`). 슬라이드 1:1 동기(SLIDE n/N).

## 1. 콘텐츠 2대 원칙 (가장 중요 — 1편 피드백의 핵심)

### 원칙 A — 2층 구조 (미래대응형)
모든 편은 두 층으로 짠다. 자료가 낡아도 뼈대(불변층)는 안 흔들리게.
- 🟦 **불변층(Foundation)**: AI 시대에 변하지 않는 원리·판단 프레임. 도구가 바뀌어도 그대로.
- 🟨 **가변층(2026.6 예시)**: 지금의 제품·가격·수치. 슬라이드 sub에 **"가변층 — 2026.6 예시 · 업데이트 대상"** 명시.
- 강사는 불변층을 **외우게**, 가변층은 **"외우지 마세요"** 라고 말한다. (이 대비가 메시지를 각인시킴)

### 원칙 B — 구체성 (추상 금지)
> ❌ "의존성이라는 보이지 않는 비용" 같은 추상은 강사가 의미를 모른다.
> ✅ 실제 숫자·제품명·계산·절차·시나리오로 내려라.

새 편 작성 시 반드시 포함:
- **실제 수치/스펙 표**(`table` 레이아웃): 장비 가격, VRAM, 단가, 사양 등 (환율 1,400원/$ 기준 환산 병기)
- **워크드 예시(계산을 끝까지 푼 것)**: "직원 N명 × … = 월 OO토큰 → 월 OO만원 → 회수 N개월" 처럼 결론 숫자까지.
- **반례/함정**: "무조건 자체구축이 싸다 → 거짓. 경량은 클라우드 승"처럼 정직하게.
- 제품명은 예시로만(곧 바뀜). 받아적게 두지 말 것.

## 2. 슬라이드 작성 표준

- **분량**: 한 편(2~3강) ≈ 20~24슬라이드. 강당 1강 ≈ 10~12슬라이드.
- **구성 흐름(권장)**: ① TITLE → ② OBJECTIVES(끝나면 할 수 있는 것 3개, 동사로) → ③ 본질 재정의(REFRAME) → ④ 원리/역사 → ⑤ 가변층 현황 → ⑥ 표/계산(구체) → ⑦ 체크리스트/프레임 → ⑧ 결론 + 실습(WORKSHOP/DO).
- **레이아웃 종류**(`lay` 키): 자동 렌더됨.
  | lay | 용도 | 데이터 키 |
  |---|---|---|
  | `title` | 강/편 표지 | (sub만) |
  | `bullets` | 일반 목록 (3~5개, 짧게) | `bullets` |
  | `compare` | 2칼럼 대비 (A vs B) | `cards=[(라벨,제목,본문,'cyan'/'purple')]` |
  | `stats` | 4카드 핵심(번호+라벨+설명) | `stats=[(번호,라벨,설명)]` |
  | `table` | 수치/스펙 표 | `cols=[...]`, `rows=[[...]]` |
- **슬라이드 텍스트는 짧게**(헤드라인 + 핵심 글머리). 상세 설명은 전부 스피커노트로.
- sub에 🟦/🟨를 붙이면 색으로 자동 구분(슬라이드에선 이모지 제거됨).

## 3. 스피커노트 작성 표준 (강의 진행의 핵심)

각 슬라이드 `notes=[(태그, 내용), ...]`. **슬라이드당 4~6블록.** 태그:

| 태그 | 뜻 | 작성 요령 |
|---|---|---|
| `SAY` | 그대로 말할 멘트 | 큰따옴표로. 실제 입에서 나오는 문장. 숫자·결론 포함. |
| `STORY` | 사례·맥락 | 구체 인물/회사/숫자로. (러닝 케이스 연결 → §4) |
| `Q` | 청중에게 던질 질문 | "손 들어보세요" 식. 예상 반응 + 받아치는 멘트까지. |
| `DO` | 실습 지시 | 인원·시간·빈칸 양식까지 구체적으로. |
| `TIP` | 강사 함정·요령 | 자주 나오는 질문과 답, 흔한 실수 차단. |
| **`NOTE`** | **강사 배경지식 (필수)** | **강사가 모를 수 있는 용어·원리를 설명. "강사가 의미를 모르면 안 된다"의 해답.** |
| `BRIDGE` | 다음 슬라이드 연결 | 점선+이탤릭 자동. 다음 질문을 던져 논리 끊김 방지. |

> **NOTE가 1편 피드백의 핵심 해결책이다.** 전문 용어(토큰·VRAM·양자화·RAG·에이전트·관련 법규 등)가
> 처음 나오는 슬라이드에는 반드시 `NOTE`로 "강사 배경지식"을 달아, 비전공 강사도 이해하고 즉답하게 한다.
> 예: VRAM 계산법 = 파라미터수 × 0.6GB(Q4), 토큰 = 한글 1자 ≈ 1.5~2토큰.

## 4. 논리·스토리텔링 (1편 피드백 반영)

- **러닝 케이스 1개를 편 전체에 관통**시킨다. 1편 = "시화공단 자동차 부품사 35명, IT담당 김 대리".
  문제 → 본질 → 우리 회사 숫자 → 결정(경계선) → 실행계획이 **한 인물의 여정**으로 이어지게.
- 편마다 같은 인물을 재등장시켜도 좋다(연속성). 새 편은 그 인물의 "다음 단계"로 시작.
- 각 슬라이드 `BRIDGE`가 **다음 슬라이드의 질문을 미리 던지게** 해 논리 흐름을 만든다.
- 결론 슬라이드엔 "정점 한 문장"을 둔다. (1편: "한 회사, 두 개의 답 — 경계선을 그어라")

## 5. 디자인/스타일 (코스1 매칭 — 손댈 필요 없음)
데이터만 채우면 아래가 자동 적용된다. (바꾸지 말 것)
- **PPTX**: 배경 `#070810` + 보라 `#8D6DF0`/시안 `#46C7EE` 그라데이션, 시안 kicker, 대형 흰 헤드라인,
  footer(`{강} · {카테고리}` 좌 / `NN / N` 우). 슬라이드별 3840×2160 PNG로 렌더 후 PPTX에 풀블리드 삽입.
- **PDF**: 화이트. 표지(중앙 헤드라인+메타블록) → 본문(헤더룰 + 요약박스 + ⏱시간줄 + 노트박스).
  SAY/STORY/Q/DO/TIP/NOTE = 실선 칩라벨 박스, BRIDGE = 점선+이탤릭.
- 폰트: NotoSansKR(한글) + NotoEmoji(흑백 이모지) `~/.fonts/`에 있음.

## 6. 제작 파이프라인 (미니PC `~/company/website/`)

```bash
cd ~/company/website
A=scripts/academy
# 1) 새 편 데이터 작성: $A/buildai_part1_data.py 복제 → buildai_part{N}_data.py 로 내용 교체
#    (META, SLIDES, 그리고 import 경로의 모듈명도 교체)
python3 $A/render_slides_html.py        # SLIDES → _buildai_part1_slides.html
node    $A/shots.js $A/_buildai_part1_slides.html $A/_shots   # 슬라이드별 PNG
python3 $A/assemble_pptx.py             # PNG → build-ai-part1-slides.pptx
python3 $A/gen_notes_html.py            # SLIDES → _buildai_part1_notes.html
node    $A/html2pdf.js "$PWD/$A/_buildai_part1_notes.html" "$PWD/src/storage/academy/build-ai-part1-speaker-notes.pdf"
```
> 2편부터는 생성기 4종(render_slides_html/shots/assemble_pptx/gen_notes_html)의 입력 모듈명·출력 키를
> `part{N}` 로 바꿔 재사용. (또는 생성기를 인자화하면 더 깔끔 — 추후 개선 과제)

검증: PNG 미리보기(Read) → PDF 페이지(Read pages) 확인 후 진행.

## 7. 배포 wiring 체크리스트 (편 공개 시)
1. `src/app/api/academy/download/[key]/route.ts` — `DOWNLOADS`에 `build-ai-part{N}-slides`(PPTX_MIME),
   `build-ai-part{N}-speaker-notes`(PDF_MIME) 2키 추가.
2. `src/app/ai-solution/academy/build-ai/page.tsx` — 해당 편 SECTIONS 항목에
   `ready: true, slidesKey, notesKey` 설정.
3. **`storage/`는 .gitignore** → 산출물은 `git add -f` 필수. 중간물 `_*`,`_shots/`는 무시됨.
4. **검증은 프로덕션 visionc.co.kr에서** (vercel.app/<hash>는 빌드별 동결). 인증→다운로드→무인증 401 확인.
5. 모든 편집·커밋은 미니PC(ubuntu@100.116.174.66)에서만.
