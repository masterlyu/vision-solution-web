### 5. 진행 프로세스

```
STEP 01 — 요구사항 분석 (1~3일)
  기능 목록, 사용자 흐름, 데이터 구조 정의.
  "이런 게 필요할 것 같아요"부터 시작해도 됩니다.

STEP 02 — 아키텍처 설계 (1주)
  기술 스택 선정, 데이터베이스 설계, API 명세서 작성.
  착수 전에 전체 설계를 보여드립니다.

STEP 03 — 스프린트 개발 (4~12주)
  2주 단위 스프린트. 진행 상황 매주 보고.
  중간 시연으로 방향 확인 후 다음 단계 진행.

STEP 04 — 배포·인수인계 (1주)
  스테이징 검증 → 프로덕션 배포 → 운영 가이드 제공.
```

> 전체 기간: **평균 6~16주** (기능·규모에 따라 상이)

### 7. FAQ

**Q1. 아직 아이디어 단계인데 상담해도 되나요?**
> 네. 대부분의 프로젝트가 아이디어 단계에서 시작합니다. "이런 기능이 있으면 좋겠다"부터 말씀해주시면 기술 검토부터 함께 해드립니다.

**Q2. 개발 중에 기능을 바꿀 수 있나요?**
> 스프린트 방식으로 진행하므로 2주마다 방향 조정이 가능합니다. 단, 이미 완료된 모듈 변경은 추가 비용이 발생할 수 있으며, 사전에 안내합니다.

**Q3. 완성 후 소스 코드를 받을 수 있나요?**
> 네. 납품 시 전체 소스 코드를 제공합니다. 이후 다른 개발자가 유지보수해도 됩니다.

**Q4. 완성 후 유지보수는 어떻게 되나요?**
> 납품 후 30일 무상 버그 수정을 제공합니다. 이후 기능 추가·유지보수는 별도 계약으로 진행합니다.

**Q5. 개발 진행 상황을 어떻게 확인할 수 있나요?**
> 2주마다 스프린트 결과물을 시연하고, 진행 현황은 프로젝트 관리 도구에서 실시간 확인 가능합니다.

**Q6. 비슷한 업종의 개발 경험이 있나요?**
> 제조·유통·서비스·의료·교육 등 다양한 업종에서 시스템을 개발했습니다. 상담 시 업종을 알려주시면 관련 사례를 공유합니다.
