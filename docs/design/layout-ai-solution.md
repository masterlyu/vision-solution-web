# 레이아웃 명세 — `/ai-solution` (AI 솔루션)

> VIS-196 산출물 | AI 디자이너 작성 | 2026-04-17

---

## 0. 디자인 방향

- **분위기**: 낭비(반복 업무·시간 손실) → 해방감(자동화·AI로 자유로워짐)
- **색상**: `--primary` 중심 + 미래적 느낌(딥 블루/인디고 계열 허용)
- **특징**: 도입 사례 비포/애프터 수치가 핵심 — 인포그래픽 스타일
- **솔루션 유형 카드**: 6개 AI 솔루션 유형을 카드로 시각화

---

| 순번 | 섹션 명 | 레이아웃 유형 |

## 2. 섹션별 상세 명세

### Section 1 — Hero

```
레이아웃: 2열
배경: 딥 다크 (#0D1117) + 인디고 그라디언트 악센트

[좌측]
  - 배지: "중소기업 AI 솔루션" (primary pill)
  - H1 (2줄):
      "직원이 하루 2시간씩 반복 업무를 하고 있다면,
       AI가 대신할 수 있습니다"
    → 폰트: 2.6rem / bold / 흰색
    → "AI가 대신할 수 있습니다" → --primary 강조
  - 서브카피 (2줄):
      "고객 문의 자동 응답, 보고서 작성—
       지금 당장 없애도 되는 반복 업무가 있다면, 100만원대부터 시작합니다."
    → 1.05rem / 밝은 회색
  - CTA 버튼:
      "무료 AI 도입 가능 영역 분석받기 →"
      → primary 배경, 흰색 텍스트, 패딩 크게

[우측]
  - SVG/Lottie: 로봇 또는 AI 뇌 아이콘, 데이터 흐름 애니메이션
  - 크기: 400×340px (모바일: 숨김)
```

### Section 3 — 도입 사례 수치 (인포그래픽)

```
레이아웃: 4행 비포/애프터 비교 카드
배경: 다크

[섹션 헤더]
  H2: "실제 도입 사례" (흰색)

[비교 카드 × 4]
  ┌────────────────────────────────────────────────┐
  │  [업종 태그]                                    │
  │  도입 전 (회색/빨간 배경):  하루 2시간 수동 응대 │
  │  →                                             │
  │  도입 후 (primary 배경):   85% 자동 처리        │
  └────────────────────────────────────────────────┘

  사례 1: 제조사(직원 30명) 고객 문의 — 2시간 수동 → AI 85% 자동
  사례 2: 온라인몰 상품 설명 — 건당 30분 → AI 1분, 월 300건
  사례 3: 서비스업 일일 리포트 — 매일 1.5시간 → 자동 생성·0분
  사례 4: 소매업 재고 집계 — 주 2회 2시간 → 실시간 대시보드

  → 수치(85%, 1분, 0분 등)는 3rem bold + --primary 강조
```

### Section 4 — AI 솔루션 유형 카드

```
레이아웃: 3×2 카드 그리드 (모바일 2×3)
배경: 흰색

[섹션 헤더]
  H2: "어떤 AI가 내 업무에 맞을까요?"

[카드 × 6]
  ┌─────────────────────────────┐
  │  🤖 아이콘 (2.5rem)          │
  │  솔루션 이름 (bold 1rem)     │
  │  결과 설명 1줄 (회색 0.9rem) │
  └─────────────────────────────┘

  1. 🤖 AI 챗봇        — "고객 응대 24시간 자동화, 직원은 복잡한 문의만"
  2. 📊 데이터 대시보드 — "매출·재고 자동 집계, 엑셀 수작업 종료"
  3. ✍️ AI 콘텐츠 생성  — "블로그·SNS·상품 설명 초안 자동 작성"
  4. 📧 자동 리포트     — "정기 보고서 AI 생성 후 이메일 자동 발송"
  5. 🔄 업무 자동화 에이전트 — "반복 업무 처리, 사내 문서 검색"
  6. 📈 예측 분석       — "수요 예측, 이탈 고객 예측, 추천 시스템"

  카드 hover: shadow 강화 + primary 상단 border
```
